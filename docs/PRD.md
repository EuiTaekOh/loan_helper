# KB 여신 상담 도우미 — PRD

## 1. 개요

| 항목 | 내용 |
|------|------|
| 제품명 | KB 여신 상담 도우미 |
| 목적 | 여신 상담원이 고객 상담 시 대출·예금 시뮬레이션과 메모를 빠르게 처리 |
| 사용자 | 은행 여신(대출) 상담원 |
| 플랫폼 | 웹 브라우저 (데스크톱·태블릿 우선, 모바일 반응형) |

## 2. 문제 정의

- 상담 중 월 상환액·총 이자·만기 금액을 손계산하거나 별도 엑셀로 확인하는 데 시간이 든다.
- 상담 내용을 종이·개인 메모에 흩어 두면 검색·이력 관리가 어렵다.
- 고객에게 보여 줄 때 읽기 쉬운 화면과 은행 업무 용어가 필요하다.

## 3. 목표

- 한 화면에서 **대출 상환 계산 → 월별 스케줄 확인 → 상담 메모 기록**까지 이어지게 한다.
- 계산은 검증 가능한 순수 함수로 분리하고, UI는 토스 스타일로 통일한다.
- 계산 결과는 참고용임을 전제로 하며, 실제 약정과의 차이를 상담원이 설명할 수 있게 한다.

## 4. 기능 요구사항

### 4.1 대출 상환 계산

- 입력: 대출원금, 연 이자율(%), 상환기간(개월)
- 방식: **원리금균등** 상환
- 출력: 월 상환액, 총 상환액, 총 이자
- 월 상환액 표시: 원 단위 정수 반올림 (`monthlyPayment()`)
- 스케줄: 회차별 상환액·원금·이자·잔액

### 4.2 예·적금 만기 계산

- 입력: 원금, 연 이자율(%), 가입기간(개월), 이자 방식(단리 / 복리·월복리)
- 출력: 만기 예상 금액, 예상 이자 수익

### 4.3 월별 상환 스케줄

- 대출 계산 결과의 `schedule` 배열을 테이블로 표시
- 금액은 `formatWon()` 형식 (`1,000,000원`)

### 4.4 상담 메모

- 메모 추가·목록 조회·키워드 검색·삭제
- 저장: 브라우저 `localStorage` (`kb-loan-consultation-memos`)
- 생성 시각 표시 (한국어 로케일)

### 4.5 공통 UI

- 헤더: 앱 제목, 다크 모드 전환
- 카드형 섹션 레이아웃, 토스 디자인 토큰·컴포넌트 클래스 사용
- 모든 사용자-facing 텍스트: 한국어 해요체

## 5. 비기능 요구사항

- 오프라인에서도 동작 (서버 불필요)
- 금융 계산 로직은 `src/lib/finance.ts`에만 위치
- Vitest로 핵심 계산·스토리지 로직 테스트
- 민감정보(주민번호, 전체 계좌번호 등) 수집·저장 금지

## 6. 범위 밖 (Non-Goals)

- KB 시스템 API·실시간 금리 연동
- 고객·직원 인증
- 상담 메모 서버 동기화·다중 기기 공유
- DSR·LTV·보증료 등 복합 여신 심사 계산 (향후 확장 시 별도 스펙)

## 7. 화면 구조

```text
[헤더] KB 여신 상담 도우미 · 다크모드
[섹션1] 금융 계산기 (탭: 대출 상환 | 예·적금 만기)
[섹션2] 월별 상환 스케줄
[섹션3] 상담 메모 (입력 · 검색 · 목록)
```
